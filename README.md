# 村庄淹没与最近断面分析桌面工具

这是一个 Windows Python 桌面程序，用于选择建筑物数据、淹没模拟时间序列和河道断面数据，自动判断某个村庄首次受淹时间，并输出最近断面的水深或水位。

## 1. 创建环境

推荐使用 conda，Windows 上 GIS 依赖会更稳：

```powershell
conda create -n flood_app python=3.10 -y
conda activate flood_app
conda install -c conda-forge geopandas rasterio rasterstats shapely pyproj pandas openpyxl pyyaml numpy -y
```

也可以尝试 pip：

```powershell
pip install -r requirements.txt
```

## 2. 运行窗口程序

```powershell
python app.py --config config.yaml
```

当前 `config.yaml` 已按本项目真实数据预填：

- 建筑物：`E:/水科院/全市建筑物白膜/建筑物.shp`
- 淹没情景：`E:/水科院/2025山洪淹没模拟/原始数据/怀柔区/24h10a`
- 断面：`E:/水科院/怀柔平谷部分沟道实测数据`
- 村庄边界：`E:/水科院/村庄边界-高程/村庄边界-高程/村庄边界.shp`

窗口默认是简洁模式。通常只需要确认或填写：

- 村名
- 模拟淹没数据文件夹
- 河道断面数据
- 村庄边界数据，如果建筑物数据里没有村名字段
- 受淹阈值

建筑物字段、村界字段、坐标系、断面参考方式和 buffer 等都放在“高级参数”里，一般不用改。输出文件夹可以不填，默认写到程序目录下的 `outputs`。

页面下方是“动画显示区”，不再显示大段日志。分析完成后会显示：

- 当前村庄参与分析的建筑物轮廓；
- 附近断面，最近断面会高亮；
- 河流水系，如果配置了 `river_network_path`，程序会自动重投影并裁剪到当前动画范围后叠加显示；
- 已裁剪的逐时刻淹没栅格动画；默认找到首次受淹后不再重复做建筑物统计，但会继续裁剪后续小时栅格用于完整动画播放；
- 当前时刻、受淹建筑物数量、最大水深/水位、首次受淹时刻。
- 状态栏会显示当前时刻村域内达到受淹阈值的面积比例，避免把 0.001m 这类极浅水深误判成整村受淹。

动画区支持交互：

- 鼠标滚轮放大/缩小；
- 鼠标左键拖动画面；
- 使用“放大 / 缩小 / 复位”按钮调整视图。
- 默认只显示达到受淹阈值的深蓝区域，避免把 0.001m 这类极浅水深误看成全村受淹；点击“显示浅水”后才会显示浅蓝浅水区。
- 点击“隐藏河流 / 显示河流”可以开关河流水系。

也可以不打开窗口，直接按配置运行：

```powershell
python app.py --config config.yaml --no-gui
```

## 2.1 运行 Web 页面

如果希望用浏览器查看更清爽的地图和动画，可以启动本地 Web 控制台：

```powershell
python web_app.py --config config.yaml --host 127.0.0.1 --port 8765
```

然后在浏览器打开：

```text
http://127.0.0.1:8765
```

Web 页面复用同一套分析代码，输入框默认读取 `config.yaml`。浏览器不能直接获取本机文件夹真实路径，所以文件选择采用“编辑路径 / 重读配置”的方式；分析结果、Excel、CSV、校正河道仍输出到 `outputs`。

Web 动画的断面标签会随时间帧更新当前断面 buffer 内最大水深/水位；绿色校正河道标签也会显示当前帧校正河道 buffer 内最大水深/水位。

## 3. 输入数据格式

建筑物数据支持：

- `shp`
- `geojson/json`
- `gpkg`
- `csv/xlsx`，需要包含 WKT 字段，或包含 X/Y 坐标字段

淹没数据支持：

- `tif/tiff`
- `out/asc` ASCII Grid

程序会自动扫描文件夹内的时间序列。若检测到 `H_01_00.out`、`H_02_00.out` 这类文件，会优先使用 `H_*.out` 作为小时水深数据，避免误读 `QX/QY` 文件。

断面数据支持：

- 常规 GIS 矢量文件：`shp/geojson/gpkg`
- `csv/xlsx` 中的 WKT 或 X/Y 坐标
- 当前项目里的 DAT 断面文件夹。DAT 行中只要包含投影坐标形式的 `x,y,z`，即可生成断面线。

窗口中的“河道断面数据（可多选）”支持：

- 一次多选多个断面文件；
- 追加一个或多个断面文件夹；
- 直接选择当前项目的 `怀柔平谷部分沟道实测数据` 总文件夹，程序会遍历其中所有可解析 DAT 断面。

配置文件中可以用 `section_path` 写一个路径，也可以用分号分隔多个路径，或使用 `section_paths` 列表。

村庄边界用于确定哪些建筑物属于当前村。如果建筑物数据本身有村名字段，程序可以直接按村名字段筛选；如果没有村名字段，就必须提供村庄边界，用村界裁剪建筑物。当前项目中，建筑物白膜的 `地名` 是区县字段，不是村名，因此九渡河这类村庄需要用村界裁剪建筑物。

## 4. 关键参数

- `village_name`：村名。
- `threshold`：受淹判断阈值，单位米。
- `time_interval_hours`：提取前序时刻断面值的间隔，默认 1 小时。
- `value_type`：水深或水位。
- `raster_crs`：可留空。若栅格没有内嵌 CRS，程序会用当前村庄建筑物/村界范围和常见投影坐标系自动匹配；匹配失败时才需要手工填写。当前 H_*.out 通常会自动推断为 `EPSG:4548`。
- `building_prefilter_field/value`：建筑物预筛字段和值。当前用于先筛 `地名=怀柔区`，减少全市白膜计算量。
- `section_crs`：可留空。若断面没有 CRS，程序优先使用已经推断出的栅格 CRS；当前 DAT 断面可随淹没栅格自动使用同一坐标系。
- `section_reference_mode`：窗口中的“最近断面参考”。默认 `first_flooded_buildings`，表示固定当前村庄，找到首次受淹时刻后，用首批受淹建筑物整体几何遍历所有断面并取最近断面。
- `max_section_distance_m`：有效最近断面的最大允许距离，默认 3000 米。若最近候选断面超过该距离，程序会在结果中标记“无有效附近断面”，不提取该远处断面值，也不会在动画里用远处断面拉大视野。
- `section_avoid_buildings`：断面裁剪后是否避让当前村庄建筑物，默认 `true`。开启后，断面仍必须经过河中，但如果继续延伸会穿过建筑物，会在建筑物前缘截断。
- `section_building_clearance_m`：避让建筑物时保留的距离，默认 2 米。
- `river_network_path`：可选河流水系路径，可以是 `.shp` 文件，也可以是包含水系文件的文件夹。当前项目预填为 `E:/水科院/全市河流水系存档/全市河流水系存档.shp`。
- `river_network_crs`：河流水系缺少 CRS 时才需要填写；当前水系 Shapefile 自带北京 1954 地方高斯坐标，可留空。
- 可选最近断面参考方式：
  - `first_flooded_buildings`：首批受淹建筑物到所有断面的最短距离，推荐用于判断受淹触发位置对应的最近断面。
  - `first_flooded_max_building`：首次受淹时水深/水位最大的那栋建筑物到所有断面的最短距离。
  - `village_boundary_or_buildings`：优先用村庄边界；没有村界时用参与分析建筑物整体。
  - `village_boundary`：只使用村庄边界。
  - `buildings_geometry`：使用参与分析建筑物整体。

## 5. 输出结果

输出到界面选择的输出文件夹：

- `淹没分析结果_村名_情景名称.xlsx`
- `淹没分析结果_村名_情景名称.csv`

Excel 包含：

- `summary`：一行汇总结果。
- `buildings`：首次受淹时刻每栋建筑物的最大值、均值和是否受淹。
- `first_flooded_buildings`：首次受淹时被淹建筑物清单。

主要字段含义：

- `是否受淹`：该村是否至少有一栋建筑物达到阈值。
- `首次受淹时间`：时间序列中首次达到阈值的时刻。
- `最近断面编号/名称/距离`：固定当前村庄，遍历所有断面，按窗口中“最近断面参考”指定的参考对象到断面的最短直线距离选取。默认参考对象是首次受淹时达到阈值的建筑物。
- `最近候选断面编号/名称`：即使超过 `max_section_distance_m`，也记录全体断面中最近的候选断面，便于核查数据是否缺少本村附近断面。
- `最近断面是否在距离阈值内`：若为“否”，说明候选断面太远，断面水深/水位不会提取。
- `首次受淹时达阈值面积比例`：首次受淹时刻，村域内水深/水位达到阈值的面积占比。
- `最近断面参考建筑物`：当参考方式为首次受淹建筑物时，记录实际用于计算距离的建筑物编号。
- `最近断面裁剪后长度_m`：用于取值的断面段长度，优先来自断面汇总表中的桩距；裁剪段必须包含“河中”，每侧先找最近的“路”，没有路则找最近的“村庄/小区”，仍没有则用断面起点或末端。
- `最近断面原本深度_m`：实测断面地形深度，按裁剪段两端较低 H85 边界高程减去河中 H85 高程计算；它不是模拟水深。
- `最近断面是否使用河中坐标锚定` / `最近断面河中坐标到断面线距离_m`：如果断面汇总表中能识别河中 X/Y，程序会优先用河中坐标锚定裁剪段。若距离很大或未锚定，说明汇总表坐标、DAT 断面线或河流水系图层可能不在同一空间位置。
- 动画中的蓝色线是外部“全市河流水系”，青绿色虚线/点是断面汇总表生成的“实测河中”。当二者不重合时，断面裁剪和原深计算以实测河中为准。
- 程序会根据实测河中点生成绿色“校正河道”，并在 `outputs` 下输出 `校正后河道_村庄_情景.geojson`。这个文件不覆盖原始水系，可用于在 GIS 软件中核查或替代局部河道线。
- `section_distances` sheet：所有断面按距离从近到远排序，便于核查“最近断面”确实来自全部候选断面。
- `最近断面在首次受淹时刻的水深或水位`：断面位置在首次受淹栅格中的值。
- `前序时刻`：首次受淹前指定小时数对应或最接近的时刻。
- `断面取值方法`：点断面取点值；线断面默认取 5 米 buffer 内最大值。

**全量校正河道**

根据所有断面汇总表生成一套校正后河道线，不修改原始水系：

```powershell
python tools/build_corrected_river_network.py --config config.yaml
```

输出位置默认是 `outputs/corrected_rivers/`：

- `校正后河道_全部断面.geojson`：按各河道横断面的河中点连成的校正河道线。
- `校正河中点_全部断面.geojson`：用于校正的每个横断面河中/河道中心点。
- `校正河道诊断_全部断面.xlsx`：每个点的来源、断面号、H85、坐标和原始表路径。
- `断面表读取诊断.xlsx`：各断面表读取情况。

**偏移规律诊断**

用实测河中点和原始全市河流水系最近点计算偏移，判断是否适合用统一平移校正无断面河道：

```powershell
python tools/analyze_river_offset.py --config config.yaml
```

输出位置默认是 `outputs/river_offset_diagnostics/`：

- `河道偏移诊断_断面河中点.xlsx`：总体、分河道和逐点偏移统计。
- `河道偏移向量_断面河中点.geojson`：从原始河道最近点指向实测河中点的偏移向量。
- `原始河道_整体平移校正_试算.geojson`：按全体样本中位 dx/dy 平移后的试算河道，不覆盖原始数据。
- `偏移校正参数.yaml`：整体平移参数和推荐意见。

如果村庄没有受淹，也会输出结果，备注中会写明“所有时刻均未达到阈值”。

## 6. 常见问题

- 找不到村名字段：在窗口中填写“建筑物村名字段”，或提供“村庄边界数据”。当前项目村庄边界的村名字段是 `Name`。如果辅助村界字段不固定，程序会先自动扫描所有字段；仍找不到时，错误信息会列出可用字段。
- 栅格缺少 CRS：程序会先自动推断；如果日志提示自动推断失败，再填写“栅格 CRS”，例如 `EPSG:4548`。
- 断面缺少 CRS：程序会优先使用栅格 CRS；如果断面和栅格不在同一坐标系，再填写“断面 CRS”。
- 输出失败：请关闭已经打开的同名 Excel 文件。程序遇到占用时会自动加时间戳另存。
- 运行慢：优先填写建筑物预筛字段和值，例如当前项目的 `地名=怀柔区`；程序会把大栅格裁剪到村庄建筑物附近再统计。
